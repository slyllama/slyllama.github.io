Jade Spring: v1.0 Development Document

It's been a year since I started working on Jade Spring, and about five months since I released it on Steam. So much has happened in the intervening months, with moving house twice and having a car stolen! Developing Jade Spring has always been a really welcome pursuit in the midst of it all, even though sometimes it's been hard to set aside enough time to devote attention to it. Most of all, seeing people live-stream and post their creations was an absolute highlight for me this year, up with working on and seeing areyoshi bring the first chapter of their webcomic, Kattabolt, to its conclusion.

Jade Spring has been in beta all this time, and now I wanna finally commit to polishing it and releasing it as the full game! Some people have been looking forward to playing the game once it exits beta, and I have kept them waiting quite a few months. I'm very interested in putting the game into a stable state so that I can approach it as a solid framework to keep adding decorations to. And I'm also looking ahead to developing a new game called Dwelt: Iron Solstice, which would be similar to Jade Spring but formed out of original designs and a little bit broader in scope. (My ultimate dream is to have Dwelt be moddable, so that everything in Jade Spring can eventually be polished and merged into Dwelt, with the garden-clearing and narrative capabilities being the same. But that's a conversation for another time!)

I've honestly been putting off the 1.0 update a little bit. I find things easy to start, and hard to maintain, because it's hard for me to not judge my work, see only the flaws in it, and want to discard a lot of it and start over. I've found that the best way of getting out of this rut is often to write out clearly a pathway for going back in, and testing and updating things -- so I thought I'd try and do that with you all! So below is a process of me working out the broad areas that need to be addressed for Jade Spring 1.0, and then breaking them down into proper tasks. Thanks for reading, and enjoy!

New decorations

This has a few additional benefits, it's a good way to pave to way for future expansions, and it will be helpful for producing new Steam store screenshots! I also have some wonderful decorations that areyoshi helped produce to add, like the railroad bridge and some Kaineng stairs. Here are my ideas for other decorations. Thorough testing of design templates

Design templates were added a little while ago, in 1.3b. They were carefully tested by anari, who did an amazing job at uncovering and helping to fix many issues with the system. Now I just need to be really sure that the system is robust and player's aren't gonna lose their layouts or any data. My plan is to perform a series of tests which need to "pass" by functioning correctly. Reminder that no design templates are overwritten on new game except for the default slot. This is the standard behaviour and should always be respected.

  1. Create a new decoration template and make it unique by clearing the map and adding new decorations. Switch back to the default template. While on the default decoration template and with some altered decorations on the map, close the game and load back in. The player's decorations should remain in the correct place, and all other design slots intact.
  2. While on the default decoration template, add some decorations, return to the main menu, and load back in. The newest changes should be intact.
  3. Alter the default decoration template. Create a new decoration template and alter it too. With that template still activated, quit the game, re-open it, and create a new save. The game should default to a fresh default template, but the previously active template should still be active.
